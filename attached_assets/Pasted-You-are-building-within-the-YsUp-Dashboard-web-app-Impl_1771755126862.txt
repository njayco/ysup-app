API ROUTES (Next.js Route Handlers)
Create the following endpoints:
1) GET /api/google/auth-url
- Generates OAuth URL with state (CSRF) tied to user session.
2) GET /api/google/callback?code=&state=
- Exchanges code for tokens, encrypts and stores in GoogleAccount, then redirects to /dashboard/productivity?connected=1
3) POST /api/google/disconnect
- Deletes GoogleAccount tokens for the user
4) GET /api/workspace/recent?type=Pad|calc|Slideshow
- Returns recent WorkspaceDoc records
5) POST /api/workspace/create
- Body: { type, title }
- Uses Google Drive API to create:
  - Doc: application/vnd.google-apps.document
  - Sheet: application/vnd.google-apps.spreadsheet
  - Slides: application/vnd.google-apps.presentation
- Stores googleFileId + googleUrl in WorkspaceDoc and returns it
6) POST /api/workspace/import
- Body: { type, googleUrl or fileId }
- Verifies access via Drive API, stores record, returns it
7) POST /api/workspace/sync/Pad
- Body: { docId, content } where content is TipTap JSON or HTML
- Converts to Docs API requests and calls documents.batchUpdate
- Minimal v1: replace entire doc body with plain text extracted from HTML
- v2: map headings, bold/italic, bullet lists
8) POST /api/workspace/sync/calc
- Body: { sheetId, range, values }
- Calls Sheets API spreadsheets.values.update with valueInputOption=USER_ENTERED
9) POST /api/workspace/sync/Slideshow
- Body: { presentationId, slidesPayload }
- Calls Slides API presentations.batchUpdate to:
  - createSlide
  - insertText into title/body placeholders
  - optionally set background color
10) GET /api/workspace/status
- Returns whether user is connected + scopes + token expiry
